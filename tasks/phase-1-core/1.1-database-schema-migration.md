#### 2.1 Users & Authentication Table

```python
import uuid
from sqlalchemy import UUID, String, DateTime, func
from sqlalchemy.orm import Mapped, mapped_column

class User(Base):
    """User authentication and authorization."""
    __tablename__ = "users"

    # Use UUID for security (prevents enumeration attacks)
    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False, index=True)
    password_hash: Mapped[str] = mapped_column(String(255), nullable=False)  # bcrypt hash
    role: Mapped[str] = mapped_column(String(20), nullable=False)  # 'editor' | 'viewer'
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    last_login: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)

    # Validation
    __table_args__ = (
        CheckConstraint("role IN ('editor', 'viewer')", name='valid_role'),
    )
```

**Design Notes**:
- UUID primary keys prevent predictable IDs
- Indexed username for fast login lookups
- Timezone-aware timestamps for audit accuracy
- Check constraint ensures data integrity at DB level

#### 2.5 Incidents Table (Replaces Operations)

```python
from decimal import Decimal

class Incident(Base):
    """Fire/emergency incident tracking (Kanban workflow)."""
    __tablename__ = "incidents"

    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    title: Mapped[str] = mapped_column(String(255), nullable=False)
    type: Mapped[str] = mapped_column(String(50), nullable=False)  # Predefined enum
    priority: Mapped[str] = mapped_column(String(20), nullable=False)
    location_address: Mapped[str | None] = mapped_column(Text, nullable=True)
    location_lat: Mapped[Decimal | None] = mapped_column(Numeric(10, 8), nullable=True)
    location_lng: Mapped[Decimal | None] = mapped_column(Numeric(11, 8), nullable=True)
    status: Mapped[str] = mapped_column(String(50), nullable=False, default='eingegangen')
    training_flag: Mapped[bool] = mapped_column(Boolean, default=False, nullable=False, index=True)
    description: Mapped[str | None] = mapped_column(Text, nullable=True)
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now(), onupdate=func.now())
    created_by: Mapped[uuid.UUID | None] = mapped_column(UUID(as_uuid=True), ForeignKey('users.id'), nullable=True)
    completed_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)

    __table_args__ = (
        CheckConstraint("type IN ('fire', 'medical', 'technical', 'hazmat', 'other')", name='valid_incident_type'),
        CheckConstraint("priority IN ('low', 'medium', 'high', 'critical')", name='valid_priority'),
        CheckConstraint("status IN ('eingegangen', 'reko', 'disponiert', 'einsatz', 'einsatz_beendet', 'abschluss')", name='valid_status'),
        CheckConstraint(
            "(location_lat IS NULL AND location_lng IS NULL) OR (location_lat IS NOT NULL AND location_lng IS NOT NULL)",
            name='valid_location'
        ),
        Index('idx_incidents_training', 'training_flag'),
        Index('idx_incidents_status', 'status'),
    )
```

**Critical Design Decisions**:
- **Decimal for coordinates**: Avoids floating-point precision issues in geocoding
- **Training flag indexed**: Fast filtering for training mode
- **Status indexed**: Optimizes Kanban column queries
- **Location constraint**: Ensures both lat/lng present or both null
- **Predefined enums**: Type-safe incident classification

#### 2.6 Incident Assignments (Many-to-Many Junction)

```python
class IncidentAssignment(Base):
    """Many-to-many assignments of resources to incidents."""
    __tablename__ = "incident_assignments"

    id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), primary_key=True, default=uuid.uuid4)
    incident_id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), ForeignKey('incidents.id', ondelete='CASCADE'), nullable=False)
    resource_type: Mapped[str] = mapped_column(String(20), nullable=False)  # 'personnel' | 'vehicle' | 'material'
    resource_id: Mapped[uuid.UUID] = mapped_column(UUID(as_uuid=True), nullable=False)  # ID from respective table
    assigned_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    assigned_by: Mapped[uuid.UUID | None] = mapped_column(UUID(as_uuid=True), ForeignKey('users.id'), nullable=True)
    unassigned_at: Mapped[datetime | None] = mapped_column(DateTime(timezone=True), nullable=True)

    __table_args__ = (
        CheckConstraint("resource_type IN ('personnel', 'vehicle', 'material')", name='valid_resource_type'),
        # Prevent duplicate active assignments
        UniqueConstraint('incident_id', 'resource_type', 'resource_id', 'unassigned_at', name='unique_active_assignment'),
        Index('idx_assignments_incident', 'incident_id'),
        Index('idx_assignments_resource', 'resource_type', 'resource_id'),
    )
```

**Design Rationale**:
- **Polymorphic resource reference**: Single table for all resource types (simplified queries)
- **Soft unassignment**: Tracks when resources are released (historical data)
- **Cascade delete**: If incident deleted, assignments auto-deleted
- **Unique constraint**: Prevents double-booking (but allows reassignment after unassignment)

## Database Best Practices Applied

### 1. UUID Primary Keys
- **Security**: Prevents ID enumeration attacks
- **Scalability**: Globally unique, enables distributed systems
- **Trade-off**: Slightly larger indexes (acceptable for this scale)

### 2. Timezone-Aware Timestamps
```python
DateTime(timezone=True)  # Always use this!
```
- Prevents timezone bugs during DST transitions
- Critical for audit logs and compliance

### 3. Check Constraints for Data Integrity
```python
CheckConstraint("status IN ('available', 'assigned', 'maintenance')")
```
- Enforces valid values at DB level (defense in depth)
- Faster than application-level validation
- Prevents bad data from any source (not just API)

### 4. Strategic Indexing
- **Single-column indexes**: For frequently filtered fields (`training_flag`, `status`)
- **Composite indexes**: For multi-column queries (`resource_type + resource_id`)
- **Unique indexes**: Enforce business rules (`username`)

### 5. Cascade Deletes
```python
ForeignKey('incidents.id', ondelete='CASCADE')
```
- Auto-cleanup of related data
- Maintains referential integrity

### 6. JSONB for Flexible Data
- Use for semi-structured data (Reko forms, audit changes)
- Queryable with PostgreSQL JSON operators
- Allows schema evolution without migrations

### 7. Soft Deletes Where Needed
- `unassigned_at` in assignments (historical tracking)
- `completed_at` in incidents (archive vs delete)
