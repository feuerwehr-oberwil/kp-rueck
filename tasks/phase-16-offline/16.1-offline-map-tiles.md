## Task 16.1: Offline Map Tiles with Automatic Fallback

**Phase**: 16 - Offline Resilience
**Priority**: P3 (Medium - but needed now)
**Estimated Time**: 12-16 hours
**Complexity**: Medium

### Overview

Implement self-hosted map tile server with automatic online/offline fallback to ensure map functionality continues when internet connectivity is unavailable. Includes map-based location picker to improve user experience for selecting incident coordinates.

### Key Requirements

- **Automatic Fallback**: Frontend tries online OSM tiles first, automatically switches to local tiles on failure
- **Manual Toggle**: Settings option to force online or offline mode
- **Geographic Coverage**: Basel-Landschaft region + surroundings (sufficient for operations area)
- **Zoom Levels**: 0-17 (building-level detail)
- **Always-On Service**: Tile server runs as part of main docker-compose stack
- **Location Picker**: Map-based coordinate selection (replaces manual lat/lng entry)
- **Status Indicator**: Clear UI indication of online/offline mode
- **Update Process**: Manual tile updates with documented instructions

```bash
# This script should be run manually when tiles need updating

mkdir -p "$TILES_DIR"
cd "$TILES_DIR"
echo "✓ Directory created: $TILES_DIR"

if [ ! -f "switzerland-latest.osm.pbf" ]; then

        "https://download.geofabrik.de/europe/switzerland-latest.osm.pbf"
    echo "✓ Download complete"
else
    echo "✓ File already exists (skipping download)"
fi

# Step 4: Download pre-generated MBTiles (recommended) OR generate locally
echo ""
echo "[4/6] Getting map tiles..."
echo ""
echo "OPTION A (Recommended): Download pre-generated MBTiles"
echo "  Visit: https://openmaptiles.com/downloads/switzerland/"
echo "  Or: https://data.maptiler.com/downloads/tileset/osm/"
echo "  Download: 'Switzerland' region as MBTiles"
echo "  Place file in: $TILES_DIR/switzerland.mbtiles"
echo ""
echo "OPTION B (Advanced): Generate tiles locally with tilemaker"
echo "  This requires tilemaker and takes 30-60 minutes"
echo "  Docker command:"
echo "    docker run -v \$(pwd):/data ghcr.io/systemed/tilemaker:master \\"
echo "      --input /data/switzerland-latest.osm.pbf \\"
echo "      --output /data/switzerland.mbtiles \\"
echo "      --bbox 7.4,47.4,7.8,47.7"
echo ""
read -p "Have you placed switzerland.mbtiles in $TILES_DIR? (y/n) " -n 1 -r
echo ""
if [[ ! $REPLY =~ ^[Yy]$ ]]; then
    echo "Please download or generate tiles first, then run this script again."
    exit 1
fi

# Step 5: Extract Basel region (optional - if full Switzerland MBTiles)
echo ""
echo "[5/6] Extracting Basel-Landschaft region..."
if command -v tile-join &> /dev/null; then
    # Use tippecanoe's tile-join to extract region
    tile-join -o "$MBTILES_OUTPUT" \
        --bbox=7.4,47.4,7.8,47.7 \
        switzerland.mbtiles
    echo "✓ Region extracted to $MBTILES_OUTPUT"
else
    echo "⚠ tile-join not found - using full Switzerland tiles"
    echo "  To extract region, install tippecanoe:"
    echo "    brew install tippecanoe  # macOS"
    echo "    Or use the full switzerland.mbtiles as basel-region.mbtiles"
    cp switzerland.mbtiles "$MBTILES_OUTPUT"
fi

# Step 6: Copy to Docker volume
echo ""
echo "[6/6] Copying tiles to Docker volume..."
docker volume create "$DOCKER_VOLUME" 2>/dev/null || true

# Copy MBTiles to volume
docker run --rm \
    -v "$DOCKER_VOLUME:/data" \
    -v "$(pwd):/source:ro" \
    alpine:latest \
    cp /source/"$MBTILES_OUTPUT" /data/"$MBTILES_OUTPUT"

echo "✓ Tiles copied to Docker volume"

# Cleanup
cd ..
echo ""
echo "═══════════════════════════════════════════════"
echo " Tile download complete!"
echo "═══════════════════════════════════════════════"
echo ""
echo "Tile file: $MBTILES_OUTPUT"
echo "Size: $(du -h "$TILES_DIR/$MBTILES_OUTPUT" | cut -f1)"
echo "Docker volume: $DOCKER_VOLUME"
echo ""
echo "Next steps:"
echo "  1. Restart the tile server: make restart-tileserver"
echo "  2. Test tiles at: http://localhost:8080"
echo "  3. Frontend will automatically use local tiles"
echo ""
echo "Clean up temp files?"
read -p "Remove $TILES_DIR? (y/n) " -n 1 -r
echo ""
if [[ $REPLY =~ ^[Yy]$ ]]; then
    rm -rf "$TILES_DIR"
    echo "✓ Cleanup complete"
fi
```

```typescript
"use client"

import { useState, useEffect } from 'react'

type TileMode = 'auto' | 'online' | 'offline'

interface MapTilesState {
  tileUrl: string
  attribution: string
  isOnline: boolean
  mode: TileMode
  setMode: (mode: TileMode) => void
}

const ONLINE_TILE_URL = 'https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png'
const LOCAL_TILE_URL = 'http://localhost:8080/styles/basic/{z}/{x}/{y}.png'

const ONLINE_ATTRIBUTION = '&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors'
const LOCAL_ATTRIBUTION = '&copy; <a href="https://www.openstreetmap.org/copyright">OpenStreetMap</a> contributors (Offline)'

export function useMapTiles(): MapTilesState {
  const [mode, setMode] = useState<TileMode>(() => {
    // Load saved preference from localStorage
    if (typeof window !== 'undefined') {
      const saved = localStorage.getItem('map-tile-mode')
      return (saved as TileMode) || 'auto'
    }
    return 'auto'
  })

  const [isOnline, setIsOnline] = useState(true)
  const [useLocal, setUseLocal] = useState(false)

  // Save mode preference
  useEffect(() => {
    if (typeof window !== 'undefined') {
      localStorage.setItem('map-tile-mode', mode)
    }
  }, [mode])

  // Check online tile availability
  useEffect(() => {
    let mounted = true
    let checkInterval: NodeJS.Timeout

    const checkOnlineStatus = async () => {
      if (mode === 'offline') {
        setUseLocal(true)
        setIsOnline(false)
        return
      }

      if (mode === 'online') {
        setUseLocal(false)
        setIsOnline(true)
        return
      }

      // Auto mode - check if online tiles are available
      try {
        const testTile = 'https://a.tile.openstreetmap.org/0/0/0.png'
        const response = await fetch(testTile, {
          method: 'HEAD',
          cache: 'no-cache',
          signal: AbortSignal.timeout(5000)
        })

        if (mounted) {
          const online = response.ok
          setIsOnline(online)
          setUseLocal(!online)
        }
      } catch (error) {
        // Network error - assume offline
        if (mounted) {
          setIsOnline(false)
          setUseLocal(true)
        }
      }
    }

    // Check immediately
    checkOnlineStatus()

    // Check periodically (every 30 seconds)
    if (mode === 'auto') {
      checkInterval = setInterval(checkOnlineStatus, 30000)
    }

    return () => {
      mounted = false
      if (checkInterval) clearInterval(checkInterval)
    }
  }, [mode])

  const tileUrl = useLocal ? LOCAL_TILE_URL : ONLINE_TILE_URL
  const attribution = useLocal ? LOCAL_ATTRIBUTION : ONLINE_ATTRIBUTION

  return {
    tileUrl,
    attribution,
    isOnline,
    mode,
    setMode
  }
}
```

```python
import httpx

@router.get("/tileserver")
async def check_tileserver():
    """Check if local tile server is available."""
    try:
        async with httpx.AsyncClient() as client:
            response = await client.get(
                "http://localhost:8080/health",
                timeout=5.0
            )
            return {
                "status": "healthy" if response.status_code == 200 else "unhealthy",
```

# Offline Map Tiles - Setup and Maintenance

## Overview

The KP Rück system includes a self-hosted map tile server to ensure map functionality during offline operations. This document explains how to set up and maintain offline map tiles.

## Initial Setup

### 1. Download Map Tiles

Run the automated download script:

```bash
bash scripts/download-tiles.sh
```

This script will:
- Download Switzerland OSM data (~300 MB)
- Guide you through obtaining MBTiles format
- Extract the Basel-Landschaft region
- Copy tiles to Docker volume

**Recommended Source**: [OpenMapTiles](https://openmaptiles.com/downloads/switzerland/)
- Free tier includes Switzerland
- Pre-generated MBTiles format
- Updated monthly

**Alternative Sources**:
- [Geofabrik Switzerland Extract](https://download.geofabrik.de/europe/switzerland.html) + tilemaker
- [MapTiler Data](https://data.maptiler.com/downloads/tileset/osm/)

The tile server starts automatically with docker-compose:

```bash
make dev  # Development mode
# or
make up   # Production mode
```

Verify tile server is running:
```bash
curl http://localhost:8080/health
```

Visit http://localhost:8080 to see the tile server preview interface.

Test a specific tile:

## Map Tile Updates

OpenStreetMap data changes daily. Update offline tiles every 3-6 months or when significant changes occur in your area.

### When to Update

- New buildings in operational area
- Road network changes
- After major infrastructure updates
- Before important exercises/events

### Update Process

1. **Download new tiles** (15-30 minutes):
   ```bash
   bash scripts/download-tiles.sh
   ```

2. **Restart tile server** (5 seconds):
   ```bash
   make restart-tileserver
   ```

   - Frontend map tiles are cached by browser

## Troubleshooting

### Tile Server Not Starting

**Check logs**:
```bash
make tileserver-logs
```

**Common issues**:
- MBTiles file missing: Run `download-tiles.sh`
- Port 8080 already in use: Check `docker ps` and stop conflicting container
- Docker volume issues: `docker volume inspect kprueck-tileserver-data`

### Map Shows "Tile Load Error"

**Diagnosis**:
1. Check tile server health: `curl http://localhost:8080/health`
2. Check if frontend can reach tile server: Test in browser DevTools Network tab
3. Verify tile URL in map component matches server port

**Solutions**:
- Restart tile server: `make restart-tileserver`
- Check Docker network: `docker network inspect kprueck-network`
- Verify tileserver-config.json is correct

### Tiles Outdated or Missing

**Update tiles**:
```bash
bash scripts/download-tiles.sh
make restart-tileserver
```

**Check tile coverage**:
- Tile server only covers Basel-Landschaft region (zoom 10-17)
- Outside this area, map will try to use online tiles (if available)
- Adjust bounding box in download script if needed

### Large File Size

**Current setup**:
- Basel region zoom 0-17: ~1-2 GB
- Full Switzerland zoom 0-17: ~3-5 GB

**Reduce size**:
- Lower max zoom (e.g., zoom 15 instead of 17): Saves ~50%
- Smaller geographic area: Edit bounding box in download script
- Use lower resolution: Trade-off with map detail

Edit `scripts/download-tiles.sh` to change coverage area:

```bash
# Current: Basel-Landschaft

```
