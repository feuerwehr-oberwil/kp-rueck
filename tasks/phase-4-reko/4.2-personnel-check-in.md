## Overview

Implement a mobile-friendly check-in interface that allows personnel to mark themselves as present on-site for an emergency. While the personnel roster in the Resources tab contains all firefighters who could theoretically respond, this system tracks who is actually present for a specific operation. Only checked-in personnel appear in the sidebar for assignment to incidents.

## Prerequisites

- ✅ Task 1.1: `personnel` table exists
- ✅ Task 2.2: Resource assignment system operational

---

## Key Design Decisions

### 1. Token-Based Access (Like Reko Forms)

The check-in page uses a **token-based system** similar to Reko forms:

- Admin clicks button next to "Neuer Einsatz" to generate check-in QR code
- QR code contains `/check-in?token=xyz` URL
- Token is validated on backend but **no login required**
- Each deployment can have its own check-in session
- Token prevents unauthorized access while keeping interface simple

**Benefits**:
- No need to manage user accounts for check-in
- QR code can be displayed at entrance on tablet/screen
- Personnel just scan and tap their name
- Completely separate from main application login

### 2. Check-In vs Availability

The system has two separate concepts:

- **Availability** (existing): General availability status of personnel (`available`, `assigned`, `unavailable`)
- **Checked-In** (new): Whether personnel is physically present on-site for current operations

**Relationship**:
- Availability is managed in the Resources tab (admin-only)
- Check-in status is self-service via token-protected page
- A person must be `available` to check in (not `unavailable` due to vacation, sick leave, etc.)
- Checked-in personnel are the only ones shown in the incident assignment sidebar

### 3. Check-In Persistence

- Check-in status persists across sessions (not reset daily)
- Individual check-out available via same mobile interface
- Check-in timestamp tracked for reporting/analytics
- **Note**: No global "check-out all" button - will handle multiple emergencies separately in future

### 4. Mobile-First Design

- Large touch-friendly buttons/checkboxes
- Simple list view with search/filter
- Token-based access (no login required)
- Works offline with sync when connection restored

```python
"""Token generation and validation for check-in forms."""
import secrets
import hashlib

def generate_checkin_token() -> str:
    """
    Generate a unique token for check-in session.

    Unlike Reko tokens which are deterministic per incident,
    check-in tokens are random to support multiple deployments.

    Returns:
        URL-safe token string
    """
    return secrets.token_urlsafe(32)

def validate_checkin_token(token: str) -> bool:
    """
    Validate check-in token format.

    In MVP, we just check format. In production, store valid tokens in database.
    """
    # Basic validation - token should be 32+ chars, alphanumeric
    return len(token) >= 20 and token.replace('-', '').replace('_', '').isalnum()
```

### 3. Create Check-In CRUD Operations

**File**: `backend/app/crud/personnel_checkin.py`

```python
"""Personnel check-in CRUD operations."""
from datetime import datetime
import uuid

from sqlalchemy import select, update
from sqlalchemy.ext.asyncio import AsyncSession

from .. import schemas
from ..models import Personnel, User
from ..services.audit import log_action
from fastapi import Request


async def get_available_personnel(
    db: AsyncSession,
    include_checked_out: bool = True,
) -> list[Personnel]:
    """
    Get personnel eligible for check-in.

    Args:
        db: Database session
        include_checked_out: If True, returns all available personnel
                           If False, only returns checked-in personnel

    Returns:
        List of personnel (excludes unavailable personnel)
    """
    query = select(Personnel).where(
        Personnel.availability != 'unavailable'
    )

    if not include_checked_out:
        query = query.where(Personnel.checked_in == True)

    query = query.order_by(Personnel.name.asc())

    result = await db.execute(query)
    return list(result.scalars().all())


async def check_in_personnel(
    db: AsyncSession,
    personnel_id: uuid.UUID,
    current_user: User | None = None,
    request: Request | None = None,
) -> Personnel | None:
    """
    Check in a person (mark as present on-site).

    Returns:
        Updated personnel record, or None if not found or ineligible
    """
    result = await db.execute(
        select(Personnel).where(Personnel.id == personnel_id)
    )
    person = result.scalar_one_or_none()

    if not person:
        return None

    # Can't check in if unavailable
    if person.availability == 'unavailable':
        raise ValueError("Cannot check in unavailable personnel")

    # Already checked in
    if person.checked_in:
        return person

    # Update check-in status
    person.checked_in = True
    person.checked_in_at = datetime.utcnow()
    person.updated_at = datetime.utcnow()

    # Log action
    if current_user and request:
        await log_action(
            db=db,
            action_type="check_in",
            resource_type="personnel",
            resource_id=person.id,
            user=current_user,
            changes={"name": person.name, "checked_in": True},
            request=request,
        )

    await db.commit()
    await db.refresh(person)

    return person


async def check_out_personnel(
    db: AsyncSession,
    personnel_id: uuid.UUID,
    current_user: User | None = None,
    request: Request | None = None,
) -> Personnel | None:
    """
    Check out a person (mark as left site).

    Returns:
        Updated personnel record, or None if not found
    """
    result = await db.execute(
        select(Personnel).where(Personnel.id == personnel_id)
    )
    person = result.scalar_one_or_none()

    if not person:
        return None

    # Already checked out
    if not person.checked_in:
        return person

    # Update check-out status
    person.checked_in = False
    person.checked_out_at = datetime.utcnow()
    person.updated_at = datetime.utcnow()

    # Log action
    if current_user and request:
        await log_action(
            db=db,
            action_type="check_out",
            resource_type="personnel",
            resource_id=person.id,
            user=current_user,
            changes={"name": person.name, "checked_in": False},
            request=request,
        )

    await db.commit()
    await db.refresh(person)

    return person


```

**Note**: Bulk check-in/out operations removed in favor of individual operations. Multiple emergency support will be added later with separate check-in sessions per emergency.

```python
"""Personnel check-in API endpoints."""
import uuid

from fastapi import APIRouter, Depends, HTTPException, Query, Request
from sqlalchemy.ext.asyncio import AsyncSession

from .. import schemas
from ..auth.dependencies import CurrentEditor
from ..crud import personnel_checkin as crud
from ..database import get_db
from ..services.tokens import generate_checkin_token, validate_checkin_token

router = APIRouter(prefix="/personnel/check-in", tags=["personnel-checkin"])


@router.post("/generate-link", response_model=dict)
async def generate_checkin_link(
    current_user: CurrentEditor = Depends(),  # Editor only
):
    """
    Generate check-in link with QR code (editor only).

    Returns shareable link and QR code data.
    Place this button next to "Neuer Einsatz" in the UI.
    """
    token = generate_checkin_token()
    link = f"/check-in?token={token}"

    return {
        "token": token,
        "link": link,
        "full_url": f"{request.base_url.rstrip('/')}{link}",
        "qr_code_data": link,  # Frontend will generate QR code from this
    }


@router.get("/list", response_model=list[schemas.PersonnelListItem])
async def list_personnel_for_checkin(
    token: str = Query(..., description="Access token from QR code"),
    checked_in_only: bool = Query(default=False, description="Only show checked-in personnel"),
    db: AsyncSession = Depends(get_db),
):
    """
    Get list of personnel for check-in interface.

    Requires valid token (no user authentication).
    Excludes unavailable personnel.
    """
    if not validate_checkin_token(token):
        raise HTTPException(status_code=401, detail="Invalid or expired token")

    personnel = await crud.get_available_personnel(
        db=db,
        include_checked_out=not checked_in_only,
    )
    return personnel


@router.post("/{personnel_id}/in", response_model=schemas.PersonnelCheckInResponse)
async def check_in(
    personnel_id: uuid.UUID,
    token: str = Query(..., description="Access token"),
    request: Request = None,
    db: AsyncSession = Depends(get_db),
):
    """
    Check in a person (mark as present).

    Requires valid token (no user authentication).
    """
    if not validate_checkin_token(token):
        raise HTTPException(status_code=401, detail="Invalid or expired token")

    try:
        person = await crud.check_in_personnel(
            db=db,
            personnel_id=personnel_id,
            current_user=None,  # No user auth
            request=request,
        )
    except ValueError as e:
        raise HTTPException(status_code=400, detail=str(e))

    if not person:
        raise HTTPException(status_code=404, detail="Personnel not found")

    return person


@router.post("/{personnel_id}/out", response_model=schemas.PersonnelCheckInResponse)
async def check_out(
    personnel_id: uuid.UUID,
    token: str = Query(..., description="Access token"),
    request: Request = None,
    db: AsyncSession = Depends(get_db),
):
    """
    Check out a person (mark as left).

    Requires valid token (no user authentication).
    """
    if not validate_checkin_token(token):
        raise HTTPException(status_code=401, detail="Invalid or expired token")

    person = await crud.check_out_personnel(
        db=db,
        personnel_id=personnel_id,
        current_user=None,  # No user auth
        request=request,
    )

    if not person:
        raise HTTPException(status_code=404, detail="Personnel not found")

    return person


@router.get("/stats", response_model=dict)
async def get_checkin_stats(
    token: str = Query(..., description="Access token"),
    db: AsyncSession = Depends(get_db),
):
    """
    Get check-in statistics.

    Requires valid token.
    Returns counts of checked-in, available, and total personnel.
    """
    if not validate_checkin_token(token):
        raise HTTPException(status_code=401, detail="Invalid or expired token")

    all_personnel = await crud.get_available_personnel(db, include_checked_out=True)
    checked_in = [p for p in all_personnel if p.checked_in]

    return {
        "total_available": len(all_personnel),
        "checked_in": len(checked_in),
        "checked_out": len(all_personnel) - len(checked_in),
    }
```

### 6. Frontend: Mobile Check-In Interface

**File**: `frontend/app/check-in/page.tsx`

```tsx
'use client';

import { useState, useEffect } from 'react';
import { useSearchParams } from 'next/navigation';
import { PersonnelListItem } from '@/lib/types';
import { apiClient } from '@/lib/api-client';
import { Input } from '@/components/ui/input';
import { CheckCircle, Circle, Search } from 'lucide-react';

export default function CheckInPage() {
  const searchParams = useSearchParams();
  const token = searchParams.get('token');

  const [personnel, setPersonnel] = useState<PersonnelListItem[]>([]);
  const [searchTerm, setSearchTerm] = useState('');
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    if (!token) {
      setError('Missing access token. Please scan the QR code.');
      setLoading(false);
      return;
    }
    loadPersonnel();
  }, [token]);

  const loadPersonnel = async () => {
    if (!token) return;

    setLoading(true);
    setError(null);
    try {
      const data = await apiClient.getCheckInList(token);
      setPersonnel(data);
    } catch (error) {
      console.error('Failed to load personnel:', error);
      setError('Invalid or expired token. Please scan the QR code again.');
    } finally {
      setLoading(false);
    }
  };

  const toggleCheckIn = async (person: PersonnelListItem) => {
    if (!token) return;

    try {
      if (person.checked_in) {
        await apiClient.checkOutPersonnel(person.id, token);
      } else {
        await apiClient.checkInPersonnel(person.id, token);
      }
      // Optimistic update
      setPersonnel(prev =>
        prev.map(p =>
          p.id === person.id ? { ...p, checked_in: !p.checked_in } : p
        )
      );
    } catch (error) {
      console.error('Check-in toggle failed:', error);
      // Reload to get correct state
      loadPersonnel();
    }
  };

  const filteredPersonnel = personnel.filter(p =>
    p.name.toLowerCase().includes(searchTerm.toLowerCase())
  );

  const stats = {
    total: personnel.length,
    checkedIn: personnel.filter(p => p.checked_in).length,
  };

  if (error) {
    return (
      <div className="min-h-screen bg-gray-50 flex items-center justify-center p-4">
        <div className="max-w-md text-center">
          <div className="text-red-600 text-xl font-semibold mb-2">
            Access Required
          </div>
          <div className="text-gray-600">{error}</div>
        </div>
      </div>
    );
  }

  return (
    <div className="min-h-screen bg-gray-50 p-4 pb-20">
      {/* Header */}
      <div className="max-w-2xl mx-auto mb-6">
        <h1 className="text-3xl font-bold text-gray-900 mb-2">
          Personnel Check-In
        </h1>
        <div className="flex gap-4 text-sm text-gray-600">
          <span>Total: {stats.total}</span>
          <span className="text-green-600 font-semibold">
            Checked In: {stats.checkedIn}
          </span>
          <span>Not Present: {stats.total - stats.checkedIn}</span>
        </div>
      </div>

      {/* Search */}
      <div className="max-w-2xl mx-auto mb-4">
        <div className="relative">
          <Search className="absolute left-3 top-1/2 -translate-y-1/2 h-5 w-5 text-gray-400" />
          <Input
            type="text"
            placeholder="Search personnel..."
            value={searchTerm}
            onChange={(e) => setSearchTerm(e.target.value)}
            className="pl-10 h-12 text-lg"
          />
        </div>
      </div>

      {/* Personnel List */}
      <div className="max-w-2xl mx-auto space-y-2">
        {loading ? (
          <div className="text-center py-8 text-gray-500">Loading...</div>
        ) : filteredPersonnel.length === 0 ? (
          <div className="text-center py-8 text-gray-500">
            No personnel found
          </div>
        ) : (
          filteredPersonnel.map(person => (
            <button
              key={person.id}
              onClick={() => toggleCheckIn(person)}
              className={`
                w-full flex items-center gap-4 p-4 rounded-lg border-2 transition-all
                ${
                  person.checked_in
                    ? 'border-green-500 bg-green-50'
                    : 'border-gray-200 bg-white hover:border-gray-300'
                }
              `}
            >
              {/* Check Icon */}
              <div className="flex-shrink-0">
                {person.checked_in ? (
                  <CheckCircle className="h-8 w-8 text-green-600" />
                ) : (
                  <Circle className="h-8 w-8 text-gray-400" />
                )}
              </div>

              {/* Info */}
              <div className="flex-1 text-left">
                <div className="font-semibold text-lg">{person.name}</div>
                {person.role && (
                  <div className="text-sm text-gray-600">{person.role}</div>
                )}
              </div>

              {/* Status Badge */}
              <div className="flex-shrink-0">
                {person.checked_in ? (
                  <span className="bg-green-600 text-white px-3 py-1 rounded-full text-sm font-semibold">
                    Present
                  </span>
                ) : (
                  <span className="bg-gray-400 text-white px-3 py-1 rounded-full text-sm">
                    Not Here
                  </span>
                )}
              </div>
            </button>
          ))
        )}
      </div>
    </div>
  );
}
```

```typescript
  return this.request(`/api/personnel${query}`);
}
```

## User Flows

### Flow 1: Admin Generates Check-In QR Code

1. Operations commander opens main dashboard
2. Clicks "Check-In QR" button next to "Neuer Einsatz"
3. System generates unique token and displays QR code dialog
4. QR code can be:
   - Displayed on tablet at entrance
   - Printed and posted at entrance
   - Shared via link
5. QR code remains valid for the session

### Flow 2: Personnel Self Check-In (Token-Based)

1. Firefighter arrives at station
2. Scans QR code with personal phone OR uses station tablet
3. Check-in page opens with `/check-in?token=xyz`
4. Searches for their name or scrolls through list
5. Taps their name to toggle check-in
6. Card turns green and shows "Present" badge
7. System updates database and they appear in assignment sidebar

### Flow 3: Gate Personnel Managing Check-Ins

1. Person at gate has check-in page open on tablet (scanned QR code)
2. As each firefighter arrives, gate person taps their name
3. Can see real-time count of checked-in personnel
4. Can check people out as they leave (tap again to toggle off)
5. No login required - just needs valid token from QR code

### Flow 4: Incident Commander Assigning Personnel

1. Opens incident card in Kanban view
2. Clicks "Assign Personnel" in sidebar
3. **Only sees checked-in personnel in the list**
4. Selects personnel and assigns to incident
5. Selected personnel's status changes to "assigned"

### Flow 5: Accessing Without Token (Error Case)

1. User tries to open `/check-in` without token parameter
2. Error message shown: "Missing access token. Please scan the QR code."
3. User must obtain valid QR code from operations commander
