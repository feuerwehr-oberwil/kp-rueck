### 1. Database Backup Script

**File**: `backend/scripts/backup_database.sh`

```bash
#!/bin/bash
# Automated database backup script
# Run daily via cron or GitHub Actions

set -e

# Configuration
BACKUP_DIR="${BACKUP_DIR:-/backups/postgres}"
RETENTION_DAYS="${RETENTION_DAYS:-30}"
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/kprueck_${TIMESTAMP}.sql.gz"

# Database connection (from environment)
DATABASE_URL="${DATABASE_URL}"

if [ -z "$DATABASE_URL" ]; then
    echo "Error: DATABASE_URL not set"
    exit 1
fi

echo "================================"
echo "KP Rück Database Backup"
echo "Time: $(date)"
echo "================================"

# Create backup directory
mkdir -p "${BACKUP_DIR}"

# Perform backup with pg_dump
echo "Creating backup..."
pg_dump "${DATABASE_URL}" | gzip > "${BACKUP_FILE}"

# Check if backup was successful
if [ $? -eq 0 ]; then
    BACKUP_SIZE=$(du -h "${BACKUP_FILE}" | cut -f1)
    echo "✓ Backup successful: ${BACKUP_FILE} (${BACKUP_SIZE})"
else
    echo "✗ Backup failed!"
    exit 1
fi

# Cleanup old backups (keep last 30 days)
echo "Cleaning up old backups (retention: ${RETENTION_DAYS} days)..."
find "${BACKUP_DIR}" -name "kprueck_*.sql.gz" -mtime +${RETENTION_DAYS} -delete
REMAINING=$(ls -1 "${BACKUP_DIR}"/kprueck_*.sql.gz 2>/dev/null | wc -l)
echo "Backups remaining: ${REMAINING}"

# Optional: Upload to S3 or external storage
if [ -n "$AWS_S3_BUCKET" ]; then
    echo "Uploading to S3: ${AWS_S3_BUCKET}"
    aws s3 cp "${BACKUP_FILE}" "s3://${AWS_S3_BUCKET}/backups/postgres/"
fi

echo "================================"
echo "Backup complete"
echo "================================"
```

### 2. Railway→Local Sync Script

**File**: `backend/scripts/sync_railway_to_local.sh`

```bash
#!/bin/bash
# Sync Railway database to local Docker deployment
# Run every 5 minutes via cron

set -e

# Configuration
RAILWAY_DB_URL="${RAILWAY_DATABASE_URL}"
LOCAL_DB_URL="${LOCAL_DATABASE_URL:-postgresql://kprueck:kprueck@localhost:5433/kprueck}"
TEMP_DUMP="/tmp/railway_sync_$(date +%s).sql"
RAILWAY_HEALTH_URL="${RAILWAY_API_URL}/api/health"

# Colors for output
GREEN='\033[0;32m'
RED='\033[0;31m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

echo "================================"
echo "Railway→Local DB Sync"
echo "Time: $(date)"
echo "================================"

# 1. Health check Railway
echo "Checking Railway health..."
if curl -sf "${RAILWAY_HEALTH_URL}" > /dev/null 2>&1; then
    echo -e "${GREEN}✓ Railway is healthy${NC}"
else
    echo -e "${RED}✗ Railway is unreachable${NC}"
    echo "Skipping sync (Railway down)"
    exit 1
fi

# 2. Dump Railway database
echo "Dumping Railway database..."
if pg_dump "${RAILWAY_DB_URL}" > "${TEMP_DUMP}"; then
    DUMP_SIZE=$(du -h "${TEMP_DUMP}" | cut -f1)
    echo -e "${GREEN}✓ Dump successful (${DUMP_SIZE})${NC}"
else
    echo -e "${RED}✗ Dump failed${NC}"
    rm -f "${TEMP_DUMP}"
    exit 1
fi

# 3. Stop local services (optional)
echo "Stopping local services..."
docker-compose -f docker-compose.local.yml stop backend frontend || true

# 4. Drop and recreate local database
echo "Recreating local database..."
docker-compose -f docker-compose.local.yml exec -T postgres psql -U kprueck -c "DROP DATABASE IF EXISTS kprueck;"
docker-compose -f docker-compose.local.yml exec -T postgres psql -U kprueck -c "CREATE DATABASE kprueck;"

# 5. Restore to local
echo "Restoring to local database..."
if psql "${LOCAL_DB_URL}" < "${TEMP_DUMP}"; then
    echo -e "${GREEN}✓ Restore successful${NC}"
else
    echo -e "${RED}✗ Restore failed${NC}"
    rm -f "${TEMP_DUMP}"
    exit 1
fi

# 6. Cleanup temp file
rm -f "${TEMP_DUMP}"

# 7. Restart local services
echo "Restarting local services..."
docker-compose -f docker-compose.local.yml start backend frontend

echo "================================"
echo -e "${GREEN}Sync complete${NC}"
echo "================================"

# 8. Log sync event
echo "$(date): Sync successful" >> /var/log/railway_sync.log
```

### 3. Restoration Script

**File**: `backend/scripts/restore_database.sh`

```bash
#!/bin/bash
# Restore database from backup

set -e

if [ $# -lt 1 ]; then
    echo "Usage: $0 <backup_file> [target_database_url]"
    echo "Example: $0 /backups/postgres/kprueck_20241024_120000.sql.gz"
    exit 1
fi

BACKUP_FILE="$1"
TARGET_DB_URL="${2:-$DATABASE_URL}"

if [ ! -f "$BACKUP_FILE" ]; then
    echo "Error: Backup file not found: $BACKUP_FILE"
    exit 1
fi

if [ -z "$TARGET_DB_URL" ]; then
    echo "Error: Target database URL not set"
    exit 1
fi

echo "================================"
echo "Database Restoration"
echo "Backup: $BACKUP_FILE"
echo "Target: ${TARGET_DB_URL%%@*}@***"
echo "================================"

read -p "This will overwrite the target database. Continue? (y/N) " -n 1 -r
echo
if [[ ! $REPLY =~ ^[Yy]$ ]]; then
    echo "Restoration cancelled"
    exit 0
fi

echo "Decompressing backup..."
if [[ "$BACKUP_FILE" == *.gz ]]; then
    gunzip -c "$BACKUP_FILE" | psql "$TARGET_DB_URL"
else
    psql "$TARGET_DB_URL" < "$BACKUP_FILE"
fi

if [ $? -eq 0 ]; then
    echo "✓ Restoration successful"
else
    echo "✗ Restoration failed"
    exit 1
fi

echo "================================"
echo "Restoration complete"
echo "================================"
```

### 4. Health Check Monitoring

**File**: `backend/scripts/monitor_railway_health.sh`

```bash
#!/bin/bash
# Monitor Railway health and alert on failures
# Run every minute via cron

HEALTH_URL="${RAILWAY_API_URL}/api/health"
LOG_FILE="/var/log/railway_health.log"
ALERT_EMAIL="${ADMIN_EMAIL:-admin@example.com}"
FAILURE_THRESHOLD=3

# Check health
if curl -sf "${HEALTH_URL}" > /dev/null 2>&1; then
    echo "$(date): HEALTHY" >> "$LOG_FILE"
    # Reset failure counter
    echo "0" > /tmp/railway_failure_count
else
    echo "$(date): UNHEALTHY" >> "$LOG_FILE"

    # Increment failure counter
    FAILURES=$(cat /tmp/railway_failure_count 2>/dev/null || echo "0")
    FAILURES=$((FAILURES + 1))
    echo "$FAILURES" > /tmp/railway_failure_count

    # Alert if threshold exceeded
    if [ "$FAILURES" -ge "$FAILURE_THRESHOLD" ]; then
        echo "Railway down for $FAILURES checks - sending alert"

        # Send email alert (requires mail configured)
        echo "Railway health check failed $FAILURES times" | \
            mail -s "ALERT: Railway Down" "$ALERT_EMAIL"

        # Reset counter after alert
        echo "0" > /tmp/railway_failure_count
    fi
fi
```

### 5. Cron Configuration

**File**: `backend/scripts/setup_cron.sh`

```bash
#!/bin/bash
# Setup cron jobs for backup and sync

CRON_FILE="/etc/cron.d/kprueck"

cat > "$CRON_FILE" << 'EOF'
# KP Rück Backup and Sync Jobs

# Daily backup at 2 AM
0 2 * * * root /app/scripts/backup_database.sh >> /var/log/kprueck_backup.log 2>&1

# Railway→Local sync every 5 minutes
*/5 * * * * root /app/scripts/sync_railway_to_local.sh >> /var/log/kprueck_sync.log 2>&1

# Health check every minute
* * * * * root /app/scripts/monitor_railway_health.sh >> /var/log/railway_health.log 2>&1

# Cleanup old logs weekly
0 3 * * 0 root find /var/log -name "kprueck_*.log" -mtime +30 -delete
EOF

chmod 644 "$CRON_FILE"
echo "Cron jobs configured"
```

### 6. GitHub Actions Backup Workflow

**File**: `.github/workflows/backup.yml`

```yaml
name: Database Backup

on:
  schedule:
    # Daily at 2:00 UTC
    - cron: '0 2 * * *'
  workflow_dispatch:  # Manual trigger

jobs:
  backup:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup PostgreSQL client
        run: |
          sudo apt-get update
          sudo apt-get install -y postgresql-client

      - name: Create backup
        env:
          DATABASE_URL: ${{ secrets.RAILWAY_DATABASE_URL }}
          BACKUP_DIR: ./backups
        run: |
          mkdir -p backups
          bash backend/scripts/backup_database.sh

      - name: Upload to S3
        if: ${{ secrets.AWS_S3_BUCKET }}
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          AWS_S3_BUCKET: ${{ secrets.AWS_S3_BUCKET }}
        run: |
          aws s3 sync backups/ s3://$AWS_S3_BUCKET/kprueck-backups/

      - name: Archive backup
        uses: actions/upload-artifact@v4
        with:
          name: database-backup-${{ github.run_number }}
          path: backups/*.sql.gz
          retention-days: 30

      - name: Notify on failure
        if: failure()
        uses: dawidd6/action-send-mail@v3
        with:
          server_address: smtp.gmail.com
          server_port: 465
          username: ${{ secrets.MAIL_USERNAME }}
          password: ${{ secrets.MAIL_PASSWORD }}
          subject: 'ALERT: KP Rück Backup Failed'
          body: 'Database backup failed. Check GitHub Actions logs.'
          to: ${{ secrets.ADMIN_EMAIL }}
```

### 7. Failover Documentation

**File**: `docs/FAILOVER_PROCEDURE.md`

```markdown
# Railway→Local Failover Procedure

## When to Failover

Failover to local deployment when:
- Railway health check fails repeatedly (>3 failures)
- Railway dashboard shows outage
- Database connection errors in production
- Scheduled Railway maintenance

## Automatic Sync Status

The local deployment automatically syncs from Railway every 5 minutes:
- Last sync time displayed in UI
- Sync logs: `/var/log/kprueck_sync.log`
- Maximum data loss: 5 minutes

## Manual Failover Steps

### 1. Verify Local Sync is Up-to-Date

```bash
# Check last sync time
tail -1 /var/log/railway_sync.log

# Verify local database is accessible
docker-compose -f docker-compose.local.yml exec postgres psql -U kprueck -d kprueck -c "SELECT COUNT(*) FROM incidents;"
```

### 2. Update DNS/Load Balancer

Point traffic to local deployment:
- **Production URL**: `https://kp-rueck.local` (or local IP)
- **Update DNS** or **reconfigure reverse proxy**

### 3. Stop Railway Sync

```bash
# Disable cron job to prevent overwriting local changes
sudo crontab -e
# Comment out the sync job
```

### 4. Notify Team

Send notification to all editors:
- "System running on LOCAL deployment"
- "Do not make changes on Railway dashboard"
- "All changes will be local until Railway is restored"

## Restore to Railway (After Outage)

### 1. Backup Local Database

```bash
bash backend/scripts/backup_database.sh
```

### 2. Push Local Changes to Railway

```bash
# Dump local database
pg_dump $LOCAL_DATABASE_URL > /tmp/local_latest.sql

# Restore to Railway
psql $RAILWAY_DATABASE_URL < /tmp/local_latest.sql
```

### 3. Re-enable Sync

```bash
# Uncomment cron job
sudo crontab -e

# Force immediate sync to verify
bash backend/scripts/sync_railway_to_local.sh
```

### 4. Switch DNS Back to Railway

Point traffic back to Railway production URL.

### 5. Monitor

Watch logs for any sync issues:
```bash
tail -f /var/log/kprueck_sync.log
```
```
