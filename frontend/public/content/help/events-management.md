# Ereignis-Verwaltung

Verwalten Sie mehrere Events, um Einsätze thematisch oder zeitlich zu trennen.

## Was ist ein Event?

Übergeordnetes Ereignis mit mehreren zugehörigen Einsätzen.

**Beispiele:**
- Grossbrand oder Unwetter (mehrere Einsätze)
- Übung (Training-Modus)
- Jahreswechsel (neue Periode)

## Events-Seite

**Zugriff:** `G` + `E` oder Navigation → "Ereignisse"

![Events Page](/help/images/events-page.png)

**Funktionen:**
- Events wechseln
- Neue Events erstellen
- Archivieren
- Exportieren

## Event erstellen

1. Events-Seite öffnen (`G` + `E`)
2. "Neues Ereignis" klicken
3. **Name** eingeben (z.B. "Unwetter 25.10.2025")
4. **Training-Flag** setzen falls Übung
5. "Erstellen"

➡️ Wechselt automatisch zum neuen Event

## Training vs. Live

**Training-Flag gesetzt:**
- Badge "Übung" wird angezeigt
- Separate Statistik
- Ideal für Schulungen

**Training-Flag nicht gesetzt:**
- Reguläre Live-Einsätze
- Erscheinen in Hauptstatistik

Das aktive Event gilt für **alle Ansichten** (Kanban, Karte, Combined).

## Archivieren & Export

**Archivieren:**
- Event-Seite → Event auswählen → "Archivieren"
- Alle Daten bleiben erhalten
- Ausgeblendet, aber wiederherstellbar

**Export:**
- Button "Export" → Format wählen (CSV/JSON)
- Enthält alle Einsätze, Ressourcen, Reko-Berichte

## Wann neues Event?

**JA:**
- ✅ Grossereignis (Unwetter, Grossbrand)
- ✅ Übung/Training
- ✅ Neue Zeitperiode (Jahreswechsel)

**NEIN:**
- ❌ Reguläre Einzeleinsätze
- ❌ Kurze Einsatzhäufung

## Häufige Fragen

**Q: Einsätze zwischen Events verschieben?**
A: Nein, Einsätze gehören zu genau einem Event.

**Q: Archivierte Events wiederherstellen?**
A: Ja, "Archivierte Events" → "Wiederherstellen"

**Q: Training-Einsätze in Statistik?**
A: Nein, werden separat geführt.
